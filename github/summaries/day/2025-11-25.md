# elizaos/eliza Daily Update (Nov 25, 2025)
## OVERVIEW 
The day focused on enhancing model parameter support with the addition of `topP` for Anthropic models and initiating a logging standardization effort. A new issue was opened regarding OpenAI-compatible API support, and an issue concerning Entity-level RLS was closed.

## KEY TECHNICAL DEVELOPMENTS

### Model Parameter Enhancements
- Introduced `topP` as a configurable model parameter, including default settings and per-model-type configurations, and ensured its pass-through in runtime-generated parameters. This enhancement specifically supports Anthropic models. ([#6166](https://github.com/elizaos/eliza/pull/6166))

## NEWLY OPENED PULL REQUESTS
- A new pull request, [#6169](https://github.com/elizaos/eliza/pull/6169), was opened to standardize logging across the Core, CLI, and Server components of the project.

## CLOSED ISSUES

### Entity-level RLS Resolution
- The issue concerning Entity-level RLS, [#6112](https://github.com/elizaos/eliza/issues/6112), which was opened on November 3rd, 2025, has been closed.
