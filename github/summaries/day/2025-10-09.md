# elizaos/eliza Daily Update (Oct 9, 2025)
## OVERVIEW 
Today, ElizaOS made significant strides in core framework development by introducing robust plugin management and configuration modules. This enhancement lays the groundwork for more modular and adaptable AI agents. A critical bug regarding import errors in the CLI was also resolved.

## KEY TECHNICAL DEVELOPMENTS

### Enhanced Core Configuration and Plugin Management
The core framework now includes advanced configuration and plugin modules, significantly improving modularity and extensibility. This update introduces automatic plugin installation, loading, validation, and dependency resolution, alongside new configuration options for characters, environments, and secrets. These changes are crucial for building more autonomous and adaptable AI agents. ([#6037](https://github.com/elizaos/eliza/pull/6037))

## CLOSED ISSUES

### Resolution of CLI Import Errors
The issue regarding "Imports not found in index.ts with Eliza CLI 1.61" ([#6031](https://github.com/elizaos/eliza/issues/6031)) has been closed. The problem was identified as missing or malformed TypeScript declaration files in version 1.6.1 of `@elizaos/core`, a build/publish pipeline issue specific to that version. Users are advised to upgrade to `@elizaos/cli@1.6.2` or later.
