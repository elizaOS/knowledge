## Weekly Newsletter (2026-05-25) — Week of **2026-05-18** to **2026-05-24**

### 1) Executive Summary

This week was a big “from framework → real-world product” moment for ElizaOS:

1. **Waifu.fun launched (the first economic agent launchpad)**  
   The community shipped **waifu.fun**, described as the first *economic agent launchpad* built with **elizaOS v3 agents**, with **Sol (The Architect)** and an Eliza agent doing the bulk of the work. It’s live, open source, and sets a concrete precedent for agent-powered launch infrastructure.  
   Discussion highlight: the launch announcement landed in Discord on May 22 and May 24.

2. **ElizaOS v2.0.2 made it to `main` (release promotion)**  
   The `develop → main` promotion for **v2.0.2** consolidated a large batch of improvements across runtime, cloud reliability, connectors, mobile/desktop stability, and build/release workflows. If you’ve been waiting for the OWNER+AGENT identity model and cloud lifecycle hardening to “feel real,” this release push is part of that stabilization story.  
   Key PR: [elizaos/eliza#7828](https://github.com/elizaos/eliza/pull/7828)

3. **The Linux/Tails-derived “milady-tails” variant was restored**  
   After prior consolidation removed the subtree, the **milady-tails** variant was restored wholesale (6k+ files) to bring back the Tails greeter UX and persistence/security properties—while acknowledging there’s still a RISC‑V gap to solve.  
   Key PR: [elizaos/eliza#7936](https://github.com/elizaos/eliza/pull/7936)

### 2) Development Updates (Technical)

Here are the technical changes that mattered most this week, grouped by theme:

#### **Identity + Connector architecture (OWNER+AGENT model)**
A lot of the ecosystem is converging on a clearer model: an agent can have its *own* identity (AGENT) while still acting on behalf of a user (OWNER) where appropriate.

- **Connector registry now supports OWNER+AGENT accounts** with schema + audit docs.  
  PRs: [#7817](https://github.com/elizaos/eliza/pull/7817), [#7819](https://github.com/elizaos/eliza/pull/7819)
- **X, Google, and Slack connectors received OWNER-side OAuth support** (dual identity flows), and Slack gained role-aware outbound routing (OWNER posts as user when configured).  
  PRs: [#7851](https://github.com/elizaos/eliza/pull/7851), [#7852](https://github.com/elizaos/eliza/pull/7852), [#7868](https://github.com/elizaos/eliza/pull/7868), [#7876](https://github.com/elizaos/eliza/pull/7876)
- **Bugfix follow-ups** smoothed UX details like per-role busy state in connector setup UI.  
  PR: [#7866](https://github.com/elizaos/eliza/pull/7866)

#### **Cloud reliability: lifecycle operations moved to a job queue**
Cloud agent lifecycle operations (restart/suspend/logs/snapshots) are increasingly treated as *asynchronous jobs* rather than inline Worker calls—critical for stability under load and for more predictable UX.

- “All agent lifecycle ops via job queue” (reducing bottlenecks and improving resilience):  
  PR: [#7810](https://github.com/elizaos/eliza/pull/7810)  
- Frontend updated to show job-tracked toasts and avoid premature “success” messages:  
  PR: [#7813](https://github.com/elizaos/eliza/pull/7813)

#### **Remote plugins: unifying “plugin” and “mode”**
A major platform direction continues: *everything is a plugin*, and plugins can run **direct** or **remote**.

- Plugin/mode unification scaffolding and runtime support landed across a sequence of PRs:  
  P0: [#7844](https://github.com/elizaos/eliza/pull/7844) • P1: [#7846](https://github.com/elizaos/eliza/pull/7846) • P2: [#7847](https://github.com/elizaos/eliza/pull/7847) • P3: [#7848](https://github.com/elizaos/eliza/pull/7848)  
  This is foundational for “agent-generated plugins,” isolated-process workers, and eventually richer agent-driven UI surfaces.

#### **Planner quality & safety: a known rough edge**
A notable new issue surfaced around chat-history search requests: the planner sometimes chooses the wrong tool (e.g., expensive `SHELL` greps) instead of `SEARCH_MESSAGES`, and can leak sub-agent control metadata into user-facing output.

- Open issue: [#7935](https://github.com/elizaos/eliza/issues/7935)  
  If you’re building assistants that must behave politely and cheaply in Discord, this one is worth tracking.

#### **On-chain identity support (ERC‑8004)**
Cloud API gained endpoints and a TS client for agent identity registration and lookup using an ERC‑8004 IdentityRegistry flow.

- PR: [#7937](https://github.com/elizaos/eliza/pull/7937)
