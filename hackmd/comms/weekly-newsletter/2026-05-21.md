# ElizaOS Weekly Newsletter (May 17 – May 23, 2026)

## 1) Executive Summary

**1) v2.0.2 shipped to `main` (release week).**  
The core repo promoted `develop → main` as **v2.0.2**, bundling a big set of runtime, UI, cloud, and packaging improvements. This release continues the push toward a more secure, identity-aware agent framework and a smoother self-host / cloud experience.

**2) OWNER + AGENT identity model landed (and is now visible in UI + connectors).**  
This week solidified the **“Owner + Agent” dual-identity architecture**: connectors can now represent *your* account (OWNER) separately from the *agent’s* account (AGENT), unlocking safer integrations with platforms like X/Twitter, Shopify, Linear, and Calendly.

**3) Hardware/OS momentum: Eliza v3 running on an emulated RISC‑V “Eliza e1”.**  
In Discord, the community celebrated **ElizaOS Debian RISC‑V 64-bit ISO booting in ~35 seconds on QEMU**, and **Eliza v3 running on ElizaOS (Linux) on the emulated “Eliza e1” chip**—a major milestone for cross-architecture validation and future hardware abstraction.

---

## 2) Development Updates (Core, Cloud, OS, Mobile)

### Identity + connectors (OWNER/AGENT)
- **Connector registry schema now supports dual accounts** (OWNER vs AGENT), with accompanying audits and tests. This is the backbone for “agent has its own identity, but you still control it.”  
- **UI updates** make these flows understandable: connector account lists can be filtered by role, and setup panels now explicitly guide OWNER vs AGENT configuration.

### Cloud reliability: agent lifecycle via job queue
Several cloud lifecycle operations (restart/suspend/logs/snapshot, etc.) moved to a **job-queue model**, reducing bottlenecks and decoupling critical agent operations from request/response paths. This improves resilience and makes failures easier to handle without taking down active sessions.

### Security + secrets: cross-platform vault
A major security milestone from earlier in the month continues paying dividends: **`@elizaos/vault`** is integrated with Settings UI for safer secrets storage (OS keychain where available, passphrase fallback for headless servers). This week’s work built on the same direction: fewer footguns for self-hosters and more consistent behavior across platforms.

### OS + packaging: Live USB hardening and cross-platform improvements
- Continued hardening of the **elizaOS Live USB** path (security manifests, health checks, packaging polish).
- **Android and Linux stability** improvements landed recently and are part of the broader v2.0.2 push (startup/connectivity fixes, local inference activation paths, TalkMode TTS behavior, etc.).
- Desktop platform work is converging toward a more modular capability architecture (see “Satellites” below).

### Platform architecture: Satellites + dynamic views
A substantial platform PR merged this week adding **first-party “Satellites”** (capability workers like FS/PTy/Git/local-model/runtime) and early **dynamic view** infrastructure. The big idea: agents can safely request capabilities and render richer UI surfaces over time without turning the main app into an unmaintainable monolith.
