# ElizaOS Weekly Newsletter (May 24–May 29, 2026)

## 1) Executive Summary

This week was a “stability + ecosystem” sprint across core runtime, Cloud, desktop/mobile, and the budding plugin economy:

1. **Shipping & hardening across the stack (Cloud → Desktop → OS):** The repo saw a wave of reliability and security work land—everything from **SOC2-oriented controls** and **security response headers** on the Cloud API to **first-run voice startup hardening** on desktop and more robust **release verification/provenance** for elizaOS Live builds.

2. **Agents got more predictable and less “meta”:** Several core fixes tightened planner/evaluator behavior to reduce phantom claims, tool-loop failure patterns, and leaking internal orchestration text—pushing agents toward **cleaner, user-facing outputs** and fewer runaway tool retries.

3. **Ecosystem expansion accelerated:** We added or matured multiple integration surfaces—**NEAR AI inference provider**, **Slack plugin migration**, improved **OWNER vs AGENT connector roles**, and community-led plugin momentum (Solana security / treasury / audit tooling), plus a big external builder signal via the **Mantle Turing Test Hackathon** announcement.

## 2) Development Updates (Core + Cloud + Desktop/Mobile + OS)

### Core runtime & agent behavior
- **Planner loop resilience improvements**
  - Better guardrails against repeated tool-arg failures and planner loops (see **#7924**).
  - Cleaner user-facing “final message” selection—restoring evaluator precedence where needed (**#7897**).
  - Prompt rules strengthened to prevent empty/bare acknowledgements and to discourage training-cutoff or internal metadata leakage (notably **#7953** and **#7965**).
- **Known issue (still open): planner quality for chat-history search**
  - **Issue #7935** reports cases where the planner chooses `SHELL` over `SEARCH_MESSAGES`, times out on large greps, and may leak sub-agent routing metadata. If you’re building chat-recall experiences, keep an eye on this thread.

### Cloud platform (elizacloud.ai)
- **Reliability & security hardening**
  - Replaced fragile HTTP control-plane health probes with a **Redis heartbeat** for better uptime and fewer false negatives (**#7899**).
  - Added **security response headers** at the Worker layer to close gaps flagged by scanning (**#7996**).
- **Critical bug fixes**
  - Fixed `/v1/chat/completions` crashes when clients pass `tool_choice: "required"` (**#7992**, cherry-picked to main as **#7993**).
  - SIWE/auth/session flow fixes and follow-ups landed earlier in the month; this week’s thread continues the push for production-safe auth behavior.
- **Identity & Web3-facing endpoints**
  - Added an **ERC-8004 IdentityRegistry** endpoint + TS client support (**#7937**), strengthening “agent identity” primitives in Cloud.

### Desktop & Mobile UX
- **Desktop first-run voice startup is sturdier**
  - A major packaged-runtime hardening pass ensured fresh installs use bundled runtime paths, staged Whisper artifacts, and correct plugin resolution (**#7994**).
- **Connector UX: OWNER vs AGENT is becoming first-class**
  - Connector registry and UI now better represent two separate identities (user vs agent) via OWNER+AGENT schema and role-aware account lists (**#7817**, **#7819**, plus follow-ups like **#7866**).
- **Automation UX (n8n) got a real clarification loop**
  - Host routes and UI now support structured clarification requests and quick-pick UI, enabling tighter workflow generation loops (**#7316**, **#7341**, plus plugin prompt/parse fixes such as **#7369**, **#7373**).

### Local inference & providers
- **New inference provider support:** **NEAR AI** joined the provider lineup (**#7883**), expanding OpenAI-compatible inference options.
- **Local inference stability improvements** continued (model routing, binary discovery, platform parity); watch for more convergence as remote-mode plugins mature.

### elizaOS Live (Linux/USB) release pipeline
- **Supply chain & verification improvements**
  - Added/expanded **SLSA provenance attestations**, SBOM attachments, checksum aggregation, and end-user verification docs (e.g., **#7909**, **#7906**, **#7902**, **#7900**, plus restores like **#7990**).
- **Multi-arch groundwork**
  - ISO builder parameterization work progressed toward **amd64/arm64/riscv64** (**#7971**) with CI matrix prep.
- **Notable milestone:** The **Tails-derived “milady-tails” variant was restored** after maintainership alignment (**#7936**), bringing back first-boot wizard UX and persistence machinery (with explicit acknowledgement that RISC‑V parity remains a known gap).
