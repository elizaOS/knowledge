# ElizaOS Weekly Newsletter (2026-05-21 → 2026-05-27)

## 1) Executive Summary

**A big “stability + shipping” week across the stack.** Here are the headline wins:

1. **Release momentum + hardened desktop first-run.** The team continued the push on recent releases (notably the develop → main promotion that triggered **v2.0.2**) and landed a major desktop reliability patch to make **first-run voice + bundled runtime startup** far less fragile (see PR **#7994**).

2. **Security & supply-chain integrity leveled up.** The repo added/strengthened SOC2-oriented controls and expanded verifiable build artifacts (SLSA provenance/SBOM work for OS deliverables shows the direction). Even when individual PRs are “plumbing,” the net outcome is simpler: **more trustworthy releases and fewer “it works on my machine” build surprises** (see **#8014**, plus OS verification docs like **#7909/#7990**).

3. **Ecosystem growth: plugins, connectors, and workflow UX.** The monorepo continues absorbing and upgrading connectors (Slack migration and role-aware auth patterns), while n8n workflow generation got a real clarification round-trip and UI to match—making automation building feel much more product-like (see **#7373**, **#7316**, **#7341**).

## 2) Development Updates

### Core runtime & agent behavior
- **Planner / tool-loop reliability improved** with multiple fixes aimed at preventing:
  - runaway tool-arg validation loops (**#7924**),
  - phantom action claims + better speaker-context preservation (**#7923**),
  - accidental leakage of internal metadata into user-facing text (a recurring theme in several orchestrator/core fixes).
- **A new ongoing quality thread**: an open issue flags that explicit “search chat history” requests can cause the planner to choose the wrong tool (e.g., **SHELL grep** timeouts) and sometimes leak sub-agent routing metadata (**Issue #7935**). If you’ve seen the bot “spin” on chat recall tasks, this is likely the same root cause.

### Desktop / Electrobun
- **First-run voice startup hardened** (bundled runtime path, staged Whisper assets, plugin resolution fixes, and safer vault bootstrap behavior) in **#7994**. This one is especially relevant if you test fresh installs or packaged builds—expect fewer “stuck at startup shell” moments.

### Cloud platform stability
- **Critical Cloud API bugfix:** chat completions no longer crash when callers pass `tool_choice: "required"` (**#7992**, and cherry-pick **#7993**). This unblocks stricter tool-use clients and planner-driven workloads.
- **Health-check architecture moved toward reliability:** control-plane liveness checks are shifting away from brittle HTTP probes to a **Redis heartbeat** mechanism (**#7899**). Net effect: fewer false negatives and fewer cascading “platform is down” symptoms when only a single component is flaky.

### OS / distribution pipeline
- A notable (and large) change: **the Tails-derived “milady-tails” variant was restored** after maintainer go-ahead (**#7936**). This brings back major UX/security primitives (Tails greeter, persistent storage machinery), with the explicit caveat that **RISC‑V parity remains a known gap** and needs a strategy.
- Verification and provenance work continues to mature:
  - end-user verification docs and helper script (**#7909**, restored via **#7990**),
  - additional supply-chain hardening patterns (SLSA provenance, SBOM attachment, ISO checks) appearing across the OS workflows.

### Connectors & plugins
- **Slack connector moved into the monorepo** (**#7375**) and is moving toward richer identity support (OWNER vs AGENT patterns are becoming a standard across connectors).
- **NEAR AI provider landed** as an OpenAI-compatible inference provider (**#7883**)—broadening provider diversity for anyone deploying agents across ecosystems.
