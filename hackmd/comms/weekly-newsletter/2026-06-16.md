## 2) Development Updates (Technical)

### Cloud Apps: Launch stabilization work
The Apps platform (per-tenant app hosting + deploy + billing) continues to tighten up:

- **Auth hardening + reliability**
  - Login surfaces were standardized on **Steward OAuth PKCE**, resolving real production login failures that previously blocked “Sign in with Eliza Cloud.”
- **Billing correctness**
  - A container-billing cron path was made **idempotent** to prevent accidental **double-debits** on retries or re-runs.
  - Organization credit defaults were changed so new orgs start at **$0** by default (preventing unintended free-credit allocation).
- **Deployment automation + “build-from-repo” path**
  - The platform can now **build user repositories directly** (Vercel-like flow), reducing the need for manual image pushes.
  - Apps deployment flags were enabled along the deployment path in staging, and automation continued around headscale + infra workflows.

Two notable blockers remain before the launch pipeline can proceed cleanly:
- **#8428 — NPM token permissions** for publishing `@elizaos/*` packages are insufficient, halting the release pipeline.
- **#8434 — Cloud 10-user launch tracking** still needs final infrastructure configuration + production flag activation.

### Model routing stability
Cloud defaults were tuned to prefer **healthy Cerebras-native defaults** in the model catalog (reducing reliance on flaky gateway suffixes and improving baseline inference stability).

### CI / DevEx improvements
- CI got smarter about cost: **label/path-gated expensive test lanes** reduce compute burn when changes don’t touch high-risk surfaces (while preserving full-strength runs for scheduled/manual workflows).
- Multiple infra workflows were improved so fewer critical operations require “operator SSH time” and more can be performed through auditable CI automation.

### Known issues from Discord (still unresolved)
A few developer-facing problems surfaced that need follow-up:
- **Windows UI blank screen** when running `bun run dev:ui` (agent starts, skills sync, but browser stays blank). Multiple users confirmed cache clears and browser swaps don’t fix it yet.
- **elizaCloud access failures** (tenant invitation requirement errors) affecting onboarding, Milady entry points, and OpenRouter settings changes.
