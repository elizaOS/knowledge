# ElizaOS Weekly Newsletter (May 31 – June 6, 2026)

## 1) Executive Summary

This week was all about **shipping cloud maturity** and **making agents more autonomous (and safer) in production**—with a side of big UX upgrades for multi-agent orchestration.

**Top milestones:**
- **Cloud V2 sync landed in production:** the team completed a major **develop → main promotion** to align production with the Cloud V2 stack and deployment workflows ([PR #8116](https://github.com/elizaos/eliza/pull/8116)).
- **Agents gained “ops-grade” lifecycle capabilities:** new foundations for **sleep/wake**, **incremental sandbox backups**, and **view-aware agent control** shipped (merged in [PR #8080](https://github.com/elizaos/eliza/pull/8080)).
- **Agent autonomy + monetization loop leveled up:** goal-driven agent loops were enabled for monetized apps (with spending caps and permissions), alongside key Cloud API routes and reliability fixes (notably [PR #8250](https://github.com/elizaos/eliza/pull/8250), [PR #8258](https://github.com/elizaos/eliza/pull/8258), [PR #8189](https://github.com/elizaos/eliza/pull/8189)).

## 2) Development Updates

### Cloud infrastructure & reliability
- **Production deploy alignment:** The Cloud V2 stack promotion to `main` triggers production deployments across backend, Cloudflare, provisioning workers, and agent images ([PR #8116](https://github.com/elizaos/eliza/pull/8116)).
- **Routing modernization:** Cloud routing moved away from BitRouter patterns, with follow-up fixes to provider routing and token pricing unit normalization (merged [PR #8233](https://github.com/elizaos/eliza/pull/8233), plus routing/pricing follow-ups like [PR #8241](https://github.com/elizaos/eliza/pull/8241) and [PR #8244](https://github.com/elizaos/eliza/pull/8244)).
- **CORS + third‑party integration readiness:** multiple changes improved browser compatibility for third-party apps and staging domains, including broader CORS handling for public APIs and extra headers used for idempotency and attribution (e.g., [PR #8245](https://github.com/elizaos/eliza/pull/8245), [PR #8252](https://github.com/elizaos/eliza/pull/8252), [PR #8196](https://github.com/elizaos/eliza/pull/8196)).

### Security hardening (Cloud auth + agent permissions)
- **PKCE + request signing for Steward OAuth:** Steward OAuth flows were hardened with PKCE and request signing, and scoped token minting was introduced (e.g., trade permissions) to reduce blast radius ([PR #8259](https://github.com/elizaos/eliza/pull/8259), [PR #8256](https://github.com/elizaos/eliza/pull/8256), [PR #8254](https://github.com/elizaos/eliza/pull/8254)).
- **Important:** A **production outage** is ongoing for “Sign in with Eliza Cloud,” caused by **OAuth 400 errors** in the Steward authorize flow. This is tracked as an open incident/bug ([Issue #8257](https://github.com/elizaos/eliza/issues/8257)) and requires coordination around signing secrets + redirect allowlists.

### Agent autonomy, orchestration & UI
- **View-agent “surface” framework shipped:** plugin views can now be made **agent-controllable** via registries/overlays and capability dispatch, with a “view-action affinity” layer that biases the planner toward actions relevant to the view the user is currently looking at ([PR #8080](https://github.com/elizaos/eliza/pull/8080)).
- **Goal-driven monetized app loop:** agents can now run a `/goal` loop for monetized applications with **capped self-spend** and economics-aware capabilities ([PR #8250](https://github.com/elizaos/eliza/pull/8250)).
- **Orchestrator UX expansion (Odysseus-style UI):** work continues on a full orchestrator workbench experience. The most complete UI implementation is currently tracked in an open PR for `plugin-task-coordinator` ([PR #8180](https://github.com/elizaos/eliza/pull/8180)) and a larger draft stack ([PR #8166](https://github.com/elizaos/eliza/pull/8166)).

### Bug fixes worth calling out
- **Reasoning model safety:** Cloud inference now guarantees a response-token budget so reasoning models don’t return `content: null` while still billing tokens ([PR #8189](https://github.com/elizaos/eliza/pull/8189)).
- **Developer experience stability:** Windows E2E and cloud mock boot reliability improved (e.g., resolving `bun.exe` execution issues: [PR #8248](https://github.com/elizaos/eliza/pull/8248)).
- **Agent runtime correctness:** multiple orchestrator and planner-loop fixes reduced duplicate replies, improved error reporting fidelity, and prevented leaked internal envelopes (e.g., [PR #8161](https://github.com/elizaos/eliza/pull/8161), [PR #8098](https://github.com/elizaos/eliza/pull/8098), [PR #8159](https://github.com/elizaos/eliza/pull/8159)).

## 4) Token Economics (AI16z + auto.fun)

No official protocol-level announcements for **AI16z token** or **auto.fun** were captured in this week’s provided logs. What *did* come through clearly:

- **Market drawdown affected sentiment:** community members openly discussed the downturn and a “survival mindset,” reinforcing the value of focusing on shipping fundamentals rather than chasing short-term price action.
- **Practical “economics plumbing” in Cloud is a priority:** the team is actively dealing with a **ledger mismatch between app-credits and org-balance** that currently blocks seamless spending. This is less “tokenomics” and more “payments correctness,” but it’s directly tied to sustainable monetization for agents.

If you’re building apps that depend on credits/spend, keep an eye on this—once resolved, it should meaningfully reduce friction for monetized agent workflows.
