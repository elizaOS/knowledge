### 2) Development Updates

Here are the most relevant engineering changes and themes from recent merged work and active threads:

**Identity & connectors (OWNER + AGENT model)**
- The **OWNER+AGENT connector architecture** continues to land across the stack—schema support, UI partitioning, and per-connector OAuth patterns.
- Recent work includes role-aware connector setup UI and connector schemas (e.g., **#7817**, **#7819**) and expanding that pattern to platforms like X and Slack (**#7851**, **#7868**, **#7876**).  
**Why it matters:** connecting “your account” (OWNER) vs “the agent’s account” (AGENT) cleanly is foundational for secure automation.

**Cloud reliability & lifecycle operations**
- Cloud agent lifecycle operations have been shifting toward **queued jobs** rather than synchronous control-plane calls (e.g., **#7810** and follow-ups like **#7813**).  
**Why it matters:** it reduces bottlenecks and makes restart/suspend/snapshot/logs more robust under load.

**Desktop/platform layer: Satellites / dynamic views / remote-mode plugins**
- The Electrobun platform work advanced with “first-party Satellites” and a dynamic view host bridge (**#7779**). In parallel, “plugin/mode unification” work has been landing (P0–P3: **#7844–#7848**), converging on a single concept: **everything is a Plugin**, with `mode: direct | remote`.  
**Why it matters:** this is the path toward safer capability separation and richer agent-created UI experiences.

**OS / Live USB track**
- A major repo event: **restoring the Tails-derived Linux variant** under `packages/os/linux/variants/milady-tails` (**#7936**). This brings back Tails’ hardened UX primitives (greeter, persistence model) and provides a strong base for privacy-oriented bootable builds.  
**Note:** multi-arch goals (including RISC-V) remain a known gap in this subtree.

**Open issue to watch: planner quality regression**
- A newly filed issue flags cases where the planner chooses **SHELL** instead of **SEARCH_MESSAGES**, times out on `grep`, and can leak sub-agent control metadata (**#7935**, open).  
**Why it matters:** tool choice reliability is core to trust—especially for Discord “search the chat” workflows.
