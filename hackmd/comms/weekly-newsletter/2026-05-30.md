## Weekly Newsletter (May 24 – May 30, 2026)

### 1) Executive Summary

This week was all about hardening the stack—across security, infrastructure reliability, and agent behavior—while the ecosystem kept expanding through new plugins and integrations.

**Top milestones:**
- **Security & release integrity leveled up (SOC2 + verification controls).** The core repo landed stronger SOC2-aligned controls and scanning/verification improvements, tightening the “ship it safely” pipeline.  
- **Cloud reliability improved with a heartbeat-based approach.** A shift away from brittle health checks toward a more reliable **Redis heartbeat** model should reduce false negatives and improve uptime during deploys and transitions.  
- **Agent output quality got cleaner and more predictable.** Planner/runtime updates continued to reduce meta-narration, duplicated task completion spam, and “phantom action” claims—making agents feel less like they’re narrating their internals and more like they’re getting work done.

---

### 2) Development Updates

#### Core framework: stability, routing, and better “tool truth”
A cluster of fixes focused on making agents behave consistently when tools are involved—especially in multi-step planner loops and sub-agent orchestration.

- **Planner + orchestration output cleanup**
  - Reduced duplicated “task complete” spam across respawned sub-agent sessions: [PR #7969](https://github.com/elizaos/eliza/pull/7969)  
  - Multiple prompt/routing rules landed to curb **empty/ack-only replies**, prevent misleading “I already did X” claims, and avoid leaking internal metadata (like training-cutoff language) into user-visible replies: e.g. [PR #7953](https://github.com/elizaos/eliza/pull/7953), [PR #7957](https://github.com/elizaos/eliza/pull/7957), [PR #7965](https://github.com/elizaos/eliza/pull/7965)

- **Open issue to watch:** chat-recall/search quality  
  There’s an active thread about the planner selecting the wrong tools (e.g., `SHELL`/`grep` timeouts) when users explicitly ask to search chat history, plus some cases where sub-agent control text leaks into user responses: [Issue #7935](https://github.com/elizaos/eliza/issues/7935)

#### Cloud: fewer brittle probes, stronger auth posture
- **Redis heartbeat replaces unstable health probes** for cloud control-plane checks: [PR #7899](https://github.com/elizaos/eliza/pull/7899)  
- **Critical API fix:** handling `tool_choice: "required"` without crashing: [PR #7992](https://github.com/elizaos/eliza/pull/7992)

#### Desktop & local inference: smoother first-run + better routing
A lot of “it works on a clean machine” polish landed—especially around voice and local inference packaging.

- **Desktop first-run voice startup hardened** (bundled runtime paths, Whisper artifacts staged correctly, improved plugin resolution): [PR #7994](https://github.com/elizaos/eliza/pull/7994)  
- **Local inference reliability improvements** including DFlash binary discovery and tighter routing/validation: [PR #7983](https://github.com/elizaos/eliza/pull/7983)

#### Ecosystem expansion: identity, inference providers, plugins
- **ERC-8004 identity registry support** (Cloud endpoint + client): [PR #7937](https://github.com/elizaos/eliza/pull/7937)  
- **NEAR AI inference provider** added as an OpenAI-compatible option: [PR #7883](https://github.com/elizaos/eliza/pull/7883)  
- Continued momentum on community plugins (notably Solana security, treasury management, compliance/audit). Several integrations are being tracked via issues:
  - Solana firewall: [Issue #7938](https://github.com/elizaos/eliza/issues/7938)  
  - Base treasury management: [Issue #7939](https://github.com/elizaos/eliza/issues/7939)  
  - Agent audit/compliance: [Issue #7974](https://github.com/elizaos/eliza/issues/7974)
