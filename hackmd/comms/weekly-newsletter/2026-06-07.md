# ElizaOS Weekly Newsletter (2026-06-07)
**Week:** 2026-06-01 → 2026-06-07

## 1) Executive Summary

This week was about **making Eliza Cloud feel “production-solid”** while continuing the push toward a more modular, composable agent ecosystem.

- **Eliza Cloud auth + onboarding got a major stability pass.** A cluster of fixes addressed session drops, OAuth redirect issues, cookie hydration problems, and passkey/MFA handling by updating the Steward SDK to **v0.10.1** and tightening the staging/prod tenant boundaries. Net effect: fewer “bounce back to login” loops and smoother mobile/CLI onboarding.

- **LifeOps was successfully decomposed into a modular “Personal Assistant” + domain plugins.** The former `plugin-lifeops` monolith was split into a new `plugin-personal-assistant` orchestrator plus **six** focused plugins (relationships, inbox, goals, blocker, finances, remote-desktop), with expanded capabilities added to existing domain plugins (todos, health, calendar, documents, phone). This is a big architectural milestone toward maintainable, independently-shippable surfaces.

- **Cloud billing/credits data integrity improved.** A long-standing ledger mismatch was resolved by unifying app-credit purchases and inference spending into a **single org-level ledger**, eliminating “stranded credits” and making monetization flows more predictable for builders.

## 2) Development Updates

### Cloud Infrastructure & Auth Reliability
A lot of engineering energy went into stabilizing the “front door” to Eliza Cloud:

- **Steward auth hardening compatibility:** fixes for PKCE/OAuth redirect behavior, cookie hydration, and token refresh sessions; plus stronger staging/prod tenant isolation to prevent staging flows from accidentally calling production endpoints.
- **Passkeys + MFA:** Steward SDK upgraded to **@stwd/sdk v0.10.1** to address passkey failures and improve MFA-required flows.
- **Mobile onboarding & CLI auth optimizations:** improvements to onboarding paths that were previously fragile on mobile browsers and CLI-driven login.

In parallel, CI/CD and deployment hygiene improved with smoke tests for the cloud mock stack and removal of a broken route verification step that was causing noisy failures.

### Plugin Architecture: LifeOps → Personal Assistant + Focused Plugins
The highlight refactor of the week:

- `plugin-lifeops` was removed (no compatibility shim) and replaced by:
  - **`plugin-personal-assistant`** (cross-domain orchestration actions like brief/prioritize/scheduled tasks)
  - New domain plugins: **relationships, inbox, goals, blocker, finances, remote-desktop**
  - Expanded/strengthened plugins: **todos, health, calendar, documents, phone**
- The UI surface moved away from a single `/lifeops` dashboard toward **per-domain views**, which should make it easier for contributors to improve one area without touching everything.

### Orchestrator & UI: Toward “Odysseus-style” Agent Workbenches
Work continues on richer operator experiences:

- A major UI effort is in progress to provide an **Odysseus-inspired orchestrator shell** (session sidebar, panels, rich composer, tool views, window management) integrated into `@elizaos/ui` and the existing orchestrator runtime. This is currently tracked as a large feature PR still under review.

### Features & Data Integrity
- **Scope-aware prompt suggestions** landed on the suggestions API endpoint (history-based, and scoped so suggestions are context-appropriate).
- **Credit ledger fix:** app-credit purchases + inference spend now reconcile cleanly under one organization-level ledger.

### CI/CD & Quality
- **Cloud mock stack smoke tests** added to catch regressions earlier.
- General cleanup of brittle checks that were blocking unrelated work.

## 4) Token Economics (AI16z, auto.fun, and Cloud Credits)

No official, week-specific announcements about **AI16z token** or **auto.fun** were captured in the provided logs. That said, several developments this week directly affect the *economics layer* of the ecosystem—especially for developers building monetized agents and apps:

- **Unified org-level credit ledger:** purchases and inference spend now reconcile in one place, reducing edge cases where credits appeared “stuck” or unusable.
