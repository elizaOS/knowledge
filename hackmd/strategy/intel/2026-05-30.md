## 3) Strategic Prioritization

### Priority 0 — Protect user trust in conversational competence (Core runtime)
**Initiative:** Fix planner/tool selection + leakage for “search chat history” intents (**#7935**).
- **User impact:** High (prevents empty replies, reduces timeouts, improves perceived “agent competence” in social/community contexts).
- **Technical risk:** Medium (touches planner prompt/tool ranking + response routing; but scoped by reproducible cases).
- **Dependencies / critical path:**
  - Ensure **Stage 1 candidateActions** (e.g., `SEARCH_MESSAGES`) are treated as a strong preference when present.
  - Add **hard safety rails**: disallow unbounded `grep -R` on repo root; enforce scope + output cap.
  - Strip/forbid sub-agent control metadata from connector-visible replies (framework-level sanitizer or stricter sub-agent output contract).
- **Recommendation (resourcing):** 1 senior engineer + 1 QA owner for 2–3 days to land guardrails + regression tests.

### Priority 1 — Reduce repetitive “closed process” support load (Token migration)
**Initiative:** Make migration closure self-serve and unmissable.
- **User impact:** High for affected users; reduces moderator load and repeated disappointment.
- **Technical risk:** Low.
- **Actions:**
  - Publish/pin a **Migration Status** post: dates, closure rationale, “no manual migrations” policy, and (if applicable) escalation policy explicitly set to “none.”
  - Add Discord **keyword autoreply** (“migrate”, “snapshot”, “portal”, “Feb 4”) pointing to the pinned post.
  - Add a short FAQ entry in docs and link it from Discord welcome/onboarding.

### Priority 2 — Capture partnership inbound without relying on ad-hoc moderator routing
**Initiative:** Standardize partner intake.
- **User impact:** Medium (ecosystem growth), but high strategic leverage.
- **Technical risk:** Low.
- **Actions:**
  - Create a single public **Partnership Contact** endpoint (email alias or form) + a short “what we partner on” page.
  - Add it to Discord channel topic and/or a pinned “Start here” message.

### Priority 3 — Improve signal-to-noise in Discord to protect support discoverability
**Initiative:** Channel taxonomy + posting guidance.
- **User impact:** Medium (support + onboarding clarity).
- **Technical risk:** Low (ops/config).
- **Actions:**
  - Split or enforce threads/tags for: **Support**, **Partnerships**, **Trading**, **Showcase/Recruiting**.
  - Encourage long-form market analysis to a dedicated channel to prevent product questions from being buried.

### Priority 4 — Convert “hackathon + recruitment” traffic into elizaOS contributions
**Initiative:** Lightweight contributor funnel.
- **User impact:** Medium (more PRs from new devs, faster iteration on UX bugs).
- **Technical risk:** Low.
- **Actions:**
  - Maintain a “Good first issues / bounties” list aligned with current pain (e.g., Discord support automation, planner tool caps, docs).
  - Provide a “Build with elizaOS in 60 minutes” starter path to leverage the Mantle hackathon audience.
