## 3) Strategic Prioritization (Impact × Risk × Dependencies)

### Priority 1 — Security guardrails for agent execution (High user impact, high risk if missed)
**Why now**
- DeFi enablement (Kamino plugin) amplifies blast radius: any signing bug, prompt injection, or boundary weakness becomes catastrophic.

**Recommended initiatives**
1. **Execution Boundary Audit Framework**
   - Define and enforce: tool permissions, signing constraints, allowlists, human confirmation points, transaction simulation requirements.
2. **Prompt-injection hardening playbook for plugins**
   - Standard patterns for: input sanitization, tool-output separation, “untrusted content” labeling, and memory write policies.

**Dependencies / critical path**
- A reusable security layer should land **before** broad DeFi action plugins mature (Kamino, Jupiter expansions, etc.).

**Resourcing**
- Allocate **1 security lead (part-time ok) + 1 platform engineer** to produce:
  - a reference implementation,
  - a plugin security checklist,
  - CI/static checks where feasible.

---

### Priority 2 — Kamino plugin as a flagship “meaningful DeFi actions” integration (High impact, medium-high technical risk)
**Why it matters**
- Community explicitly frames it as bridging agents from “info retrieval” to “real actions.”

**Risk controls (must-have)**
- Safe abstractions for:
  - collateral/health factor monitoring,
  - borrow limits and liquidation risk,
  - vault deposit/withdraw with slippage/price impact protections,
  - limit order safety (price bounds).
- Strong defaults: “read-only” mode, simulated execution, and explicit confirmation gates for signing.

**Plan (recommended)**
- Phase 0: **Read-only + portfolio/risk reporting** (fast win, low risk).
- Phase 1: **Deposits/withdrawals with strict constraints**.
- Phase 2: **Borrow/repay + advanced vault management**, only after boundary/audit framework is in place.

### Priority 3 — Contributor/bounty conversion pipeline (Medium-high impact, low technical risk)
**Observed pain**
- Multiple developers and bounty partners are asking “where do I post / who do I talk to,” and requests are going unanswered.

**Actions**
- Stand up a **#bounties-and-jobs (read-only curated)** channel or a pinned “weekly bounty thread.”
- Add a **“Start here to contribute”** pinned message:
  - current priorities,
  - plugin dev workflow (fork + develop + rebase cadence),
  - where to propose integrations,
  - who to contact for partner postings.
- Assign a rotating **triage responder** to acknowledge posts within 24h and route to owners.

**Success metrics**
- Response time to contributor/partner asks (median).
- # of bounty postings published.
- # of first-time contributors who ship a PR within 7 days of joining.
