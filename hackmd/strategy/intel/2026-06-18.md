## 3) Strategic Prioritization (Impact vs Risk, Dependencies, Resourcing)

### Priority 0 — Trust & Safety + “Official Answers” (lowest engineering risk, highest community impact)
**Initiative A: Ship a canonical “Migration Status & Safety” bulletin (pinned + bot reply)**
- **What:** A single authoritative message:
  - Migration window status (closed/open)
  - Whether any late-migration path exists (yes/no) and, if yes, exact process
  - Official links list + “no DMs” policy + scam reporting steps
- **Why now:** Migration confusion is directly generating scam attempts and repeated questions.
- **Technical risk:** Low (docs + automod + pinned posts).
- **Dependencies:** Decide policy on late migration (finality vs appeal).

**Initiative B: Anti-scam ops hardening**
- **What to change:**
  - Restore a lightweight mod workflow channel (or equivalent private mod thread) to reduce main-channel tagging noise.
  - Add automod rules for common scam patterns (airdrop claims, wallet purchase solicitations, “migration link” phrasing).
  - Standardize a “report + delete + ban” runbook for minimods.
- **Impact:** Reduces user harm + lowers moderator cognitive load.
- **Technical risk:** Low.

**Resource recommendation:** 1 Community Ops owner + 1 Moderator lead for 1 day to implement + publish.
