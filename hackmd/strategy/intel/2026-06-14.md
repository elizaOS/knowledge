## 2) User Experience Intelligence (Feedback → Themes, Impact, Opportunities)

### A. Cloud onboarding & access (High impact, immediate conversion blocker)
- **elizacloud signup error:** `"Tenant 'elizacloud' requires an invitation to join."`
  - User impact: prevents new users from even entering the product funnel.
  - Status: escalated; resolution not confirmed in provided data.
**Opportunity:** treat as **P0 onboarding SLO breach** with a public status update once fixed (reduces repeated tickets + distrust).

### B. Versioning / branch UX mismatch (Medium-high impact, recurring dev friction)
- Users report being **stuck on v1.7.2**; guidance is “use develop for 2.x”.
- Root cause is not just user error—it's **implicit release policy**:
  - develop = active 2.x
  - main = lagging/stable, but users expect “main = latest”
**Opportunity:** tighten the “intended design vs reality” gap:
- Make upgrade path explicit in CLI/docs and in-repo README.
- Provide a single “recommended install channel” per persona (builders vs production).

### C. World models adoption (Medium impact now, high strategic upside)
- Community guidance is: use **Google Vertex + Genie** now; ElizaOS tooling expected “~1 month”.
- Adoption friction is documentation/tooling maturity, not interest.
**Opportunity:** pre-release a “minimum viable world-model starter”:
- A canonical recipe + cost envelope + evaluation harness (even before full tooling ships).

### D. Trust, safety, and channel quality (High community health impact)
- Scam bots/bad actors noted in discussion channel.
- Promotional posts (PumpFun/DeFi collab) and link spam dilute signal.
**Opportunity:** shift moderation from reactive to systemic:
- Add anti-spam automations; tighten posting permissions in high-visibility channels; create a clear promotions lane.

### E. Governance/financial narrative (High reputational risk)
- Detailed fact-check post alleges circular fund flow / exit-liquidity dynamics.
- Even if inaccurate, this is now a **dominant narrative thread**.
**Opportunity:** rapid, structured response framework:
- “What we can disclose” FAQ + governance process + conflict-of-interest policy statement.
