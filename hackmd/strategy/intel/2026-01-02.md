### 3) Strategic Prioritization (Impact × Risk × Dependencies)

#### Priority A — Token & Migration Clarity (Immediate, High Impact, Low–Med Tech Risk)
**Why now**
- Multiple unanswered questions during peak attention; trust risk is compounding.
**Deliverables (48–72h)**
- Publish and pin a **Token & Migration FAQ** that explicitly includes:
  - contract addresses (per chain)
  - migration steps + deadline (or “TBD with next update date” if not final)
  - explanation of **“token integrated into cloud”** (utility, gating, payments, or roadmap intent)
  - troubleshooting (wallet issues, Phantom/MetaMask guidance)
- Add a Discord **auto-reply bot** (or mod macro) that detects “contract / CA / migrate / deadline” and links the canonical FAQ.
**Success metrics**
- Reduce repeated token questions in #discussion by **>50%** week-over-week.
- Increase migration completion confirmations (if tracked) or reduce migration-help churn.

#### Priority B — Ship a “Playable Agents” Gaming MVP Slice (High Impact, Med Tech Risk)
**Scope control**
- The goal is *not* a full game; it’s a **demoable agent loop**.
**Critical path (from core-dev action items)**
1) **Agent Mode in RuneScape fork**: user creates an agent, sees it operate in-world.
2) **Quest generation + NPC setup** in Hyperscape asset forge.
3) **World viewing/tweaking** tools before shipping (editor/inspection loop).
**Recommendation**
- Pick **one** environment for the first 2–3 week milestone:
  - Either “RuneScape fork: agent mode demo” *or*
  - “Hyperscape: asset forge quest+NPC tooling demo”
- Publish a short spec defining:
  - minimum viable behaviors (idle, navigate, interact, quest offer/accept/complete)
  - interfaces needed from Eliza agents / Spartan
  - telemetry requirements (to learn what users do)
**Success metrics**
- 1 internally repeatable demo flow (setup → spawn agents → generate quest → observe completion) under **10 minutes**.
- At least **1 external-facing clip** demonstrating agent-driven gameplay loop.

#### Priority C — Plugin Discoverability & “Can I do X?” Capability Map (Medium Impact, Low Risk)
**Problem**
- Users ask “is there a Discord plugin that works” and “can Eliza schedule Farcaster posts” because the capability surface isn’t discoverable.
**Deliverables**
- A single “Capabilities” page: **supported transports (Discord/Telegram/Web), social (X/Farcaster), automation**, and “how to enable.”
- In Discord, replace “go to #bot-commands” with a pinned **compatibility matrix** (plugin name → version → status → example config).
**Success metrics**
- Fewer repeated compatibility questions; increased successful first-run setups (proxy: fewer “does this work?” messages, more “it worked” messages).
