## 3) Strategic Prioritization (Impact × Risk × Dependencies)

### Critical path (next 7–14 days) — Cloud Apps go-live readiness
**P0: Billing leak closure (highest priority)**
- **Why:** revenue integrity + user trust; leaks create unbounded liability.
- **Actionable next steps:**
  - Convert “four billing leaks” into **tracked issues with repro + invariant definitions**:
    - expected billable units, metering boundaries (per-container runtime, per-tenant DB), and idempotency guarantees.
  - Add **automated regression tests** for billing invariants (cron re-run safety, duplicate protection, tenant isolation billing boundaries).
  - Require a **go-live checklist gate**: “0 known billing leaks” + “billing cron idempotency proven.”

**P0: Provisioning lifecycle & cloud-init completeness**
- **Why:** operational stability; missing cloud-init pieces cause drift, manual hotfixing, and delayed recovery.
- **Actionable next steps:**
  - Define a single “apps node readiness probe” script (Caddy installed/configured, pgbouncer present if required, firewall rules, provisioning agent healthy).
  - Block production deployment until probe passes on clean-provisioned nodes.

**P1: Scale guardrails (limit blast radius)**
- Ship with explicit limits:
  - max apps per node / max tenants per cluster
  - connection caps and pooler requirements
- Add dashboards/alerts: DB connections, container counts, port usage, Caddy route counts.

### Voice/Discord reliability program (companion adoption driver)
**P1: Validate @elizaos/plugin-discord voice under load**
- **Why:** companion developers explicitly seek voice; failures here are highly visible.
- **Recommended approach:**
  - Treat the external “audit” as input, but move to **internal reproducible load tests**:
    - define target concurrency (channels, speakers, events/sec)
    - measure text sync lag, dropped events, CPU/mem
  - Produce a **voice subsystem “known limits + tuning guide”** for builders (buffer sizes, backpressure behavior, recommended hardware).

**Risk note:** accepting external contractor proposals without scoped deliverables increases coordination cost. Prefer a paid trial with concrete success metrics (below).

### Community operations (protect developer signal)
**P2: Contain solicitations; improve contributor routing**
- Add/enable:
  - `#collab-hiring` (or similar) channel and a rule to redirect offers there
  - a lightweight template for “seeking collaborator” posts (project, repo link, scope, timeline)
- Publish a weekly “Help wanted” digest: top 5 issues, plugin areas needing owners (voice, docs, MCP examples).
