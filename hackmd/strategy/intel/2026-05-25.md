## 3) Strategic Prioritization (Impact × Risk × Dependency)

### Priority 0 (P0): Fix planner routing + metadata leakage (Issue #7935)
**Why now**
- Directly affects credibility of agents in public channels; failures are expensive (multi-iteration tool calls, timeouts) and confusing (empty replies).
- Blocks safe scaling of “agents as operators” (especially with wallets), because misrouting to SHELL increases blast radius.

**Recommended actions (1–2 engineers, 1–2 days for first patch)**
1. **Hard preference rule:** when Stage 1 provides `candidateActions`, require planner to pick from them unless it can justify deviation (logged).
2. **SHELL guardrails:** ban `grep -R` at repo root; require scoped paths and output caps; enforce shorter default timeouts for file-system search.
3. **Sub-agent output contract:** strip/forbid control metadata in connector-facing output; add regression tests for “Routing Kind” leakage.
4. **Action schema nudges:** update planner prompt/examples to stop emitting deprecated `TASKS { action: "spawn_agent" }`.

**Success metrics**
- ≥80% reduction in SHELL tool calls for “search messages” intents.
- 0 occurrences of “Routing Kind” strings in user-visible replies (test + telemetry).
- Median tool-latency for chat-search flows < 5s (no timeouts).
