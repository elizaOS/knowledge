## 3) Bug Fixes (critical)

### Vault/keyring segfault on headless Linux
**Symptom:** agent boot hard-crashed (native segfault) when `@napi-rs/keyring` attempted to talk to Secret Service over DBus on headless hosts.  
**Fix:** skip OS keychain on headless Linux and fall back to passphrase master key chain.

- https://github.com/elizaos/eliza/pull/7230

**Recommended server config (example):**
```bash
# headless-friendly vault master key
export MILADY_VAULT_PASSPHRASE="use-a-strong-passphrase"
```

---

### Android local inference activation not triggered by `/api/local-inference/active`
**Symptom:** stock Android APK with `ELIZA_LOCAL_LLAMA=1` would not activate the in-process native loader, leading to ‚Äúlocal inference enabled but not actually running‚Äù behavior.  
**Fix:** route now activates the Android native loader plugin in-process.

- https://github.com/elizaos/eliza/pull/7769

---

### Streaming TTS dropped voice/model hints (local inference)
**Symptom:** streaming plugin‚Äôs local TTS route reduced requests to `{ text }`, losing `voice`, `model`, `speed`, etc., producing incorrect voices and inconsistent playback.  
**Fix:** forward full hint payload.

- `fix(streaming): forward local TTS voice hints` ‚Äî https://github.com/elizaos/eliza/pull/7767

---

### Android direct-chat path capped generations too aggressively
**Symptom:** Android ‚Äúdirect local‚Äù fast path optimized for short voice replies could truncate long text generations.  
**Fix:** explicit long-generation mode added while preserving default short caps.

- `fix(agent): allow long Android direct local generations` ‚Äî https://github.com/elizaos/eliza/pull/7765

---

### Automations clarification resolution: free-form clarifications
**Symptom:** clarification resolve flow assumed every clarification maps to a `paramPath`; free-form clarifications failed.  
**Fix:** tolerate free-form clarifications in resolve flow.

- `fix(automations): support free-form clarifications in resolve flow` ‚Äî https://github.com/elizaos/eliza/pull/7370

## 7) Breaking Changes / Migration Notes (V1 ‚Üí V2)

### ai16z ‚Üí ElizaOS migration support gap (community-reported)
A user reported missing the migration deadline and being unable to reach admins for assistance. This remained unresolved in Discord discussion:
- Discord thread (discussion channel): https://discord.com/channels/1253563208833433701/1253563209462448241

**Action for developers:** if you still have ai16z-era configs/plugins, expect manual remediation:
- audit plugin IDs vs monorepo entries (collisions have occurred; e.g., polymarket ID collision fixed in `fix(registry): rename plugin polymarket id...` ‚Äî https://github.com/elizaos/eliza/pull/7314)
- verify ESM-first plugin builds (many plugins migrated earlier in May)
- confirm secrets storage: plaintext `config.env` flows are increasingly superseded by vault refs

### Secrets storage behavior changes
With vault mirroring/reveal now vault-first, any tooling that assumed secrets are readable from `config.env` may break or show placeholder/sentinel values.

Related fix preventing vault sentinels from polluting runtime env:
- `fix(agent): skip vault sentinels in applyConfigEnvToProcessEnv` ‚Äî https://github.com/elizaos/eliza/pull/7368

### Android local inference defaults changed (DFlash opt-in)
Stock APK builds no longer auto-enable DFlash just because the shim is present:
- `fix(android): keep DFlash opt-in for stock APK` ‚Äî https://github.com/elizaos/eliza/pull/7771

If you relied on auto-on speculative decoding in Android builds, you must explicitly enable it in your configuration.
