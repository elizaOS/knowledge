## 3) Bug Fixes (critical fixes + technical context)

### Docker health-check wedge from circular symlinks
**Symptom:** fresh sandboxes on `develop` intermittently failed health checks; container booted but never reached HTTP listen.  
**Root cause:** circular symlink interactions during package linking/pruning involving crypto/auth dependencies (`jose`, `@node-rs/argon2`) caused the environment to get into a pathological state.  
**Fix:** break/avoid the circular symlink behavior in the pruner/linking path; update related scripts and runtime touchpoints.

- PR: https://github.com/elizaos/eliza/pull/7783

### Windows execution reliability for LifeOps actions
**Symptom:** on Windows hosts, agents would plan actions that only exist on macOS and fail mid-run, producing poor UX and sometimes cascading planner retries.  
**Fix:** explicit platform guards and fallbacks in `plugin-lifeops`, ensuring the agent runtime can reason about what’s actionable on the current OS *before* committing to tool execution.

- PR: https://github.com/elizaos/eliza/pull/7788

## 7) Breaking Changes (V1 → V2 migration warnings)

### Migration support gap surfaced in community
Discord reports indicate at least one developer missed an **ai16z → ElizaOS** migration deadline and could not reach admins via email (unresolved in channel). If you are migrating older projects:

- Treat connector/plugin IDs, workspace layout, and config loading as potentially changed from older forks.
- File a GitHub issue with:
  - previous repo/version (ai16z commit/tag),
  - ElizaOS target version,
  - failing command + logs,
  - your `eliza.config` / environment strategy.

### Forward-looking breaking surface: connector capability refactor (pending)
If https://github.com/elizaos/eliza/pull/7782 merges, connector maintainers may need updates:
- connectors that don’t implement thread/edit/typing capabilities must declare that clearly, or orchestration/UI logic may attempt to use unavailable features.
- orchestration plugins may start routing progress/narration differently based on capabilities (thread vs edit-in-place vs send).

Track and test against: https://github.com/elizaos/eliza/pull/7782
