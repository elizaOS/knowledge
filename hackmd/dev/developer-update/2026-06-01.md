  PR: https://github.com/elizaos/eliza/pull/8104

## 2) New Features

### `requireLogicNodes()` helper for agent registry validation
A new helper was added to enforce presence of required “logic nodes” (useful for preventing partially-registered agent graphs from running in production).

- PR: https://github.com/elizaos/eliza/pull/8022

Example usage pattern (conceptual; adapt to your registry module layout):

```ts
import { requireLogicNodes } from "@elizaos/eliza/agent/registry";

const nodes = requireLogicNodes(["planner", "router", "memory"]);

// If this returns, your agent graph has the required nodes
export const myAgent = {
  nodes,
  // ...other agent config
};
```

**Why it matters:** this moves a class of failures from runtime (“missing node causes null routing / silent fallback”) to startup-time validation, which is especially important now that containerized provisioning and blue/green rollouts are becoming standard.

### Out-of-box whitelabeling
ElizaOS added support for “whitelabeling” defaults so downstream distributors can ship branded builds with fewer patches.

- PR: https://github.com/elizaos/eliza/pull/8074

### Ecosystem: MCP-based DeFi decision oracle (community integration path)
A notable builder introduction this week: **Arsenal Quant** announced the **Arsenal Decision Engine**, positioned as a high-performance “decision oracle” for DeFi agents that integrates with ElizaOS via **MCP (Model Context Protocol)** and supports **L402 Lightning** for payments.

- Discord context (builder intro): https://discord.com/channels/1253563208833433701/1253563209462448241

This is not a merged core plugin yet, but it is an important signal for **paid tools via MCP** and “agent economy” style integrations.

## 3) Bug Fixes (with technical context)

### Security hardening: destructive actions + MCP spawning + web functionality
A large security-focused merge tightened protections around:
- **destructive actions** (guardrails to prevent accidental/unauthorized operations),
- **MCP spawning** (reducing risk when launching MCP-connected tools/services),
- **web functionality** (hardening for web-exposed surfaces).

- PR: https://github.com/elizaos/eliza/pull/8076

**Developer impact:** if you maintain plugins that execute filesystem/network/process actions, validate that your action registration and permission expectations still hold after this change. Treat this as “security semantics tightened”; audit any internal tools that relied on permissive defaults.

### Reply integrity + routing correctness fixes
- Preserving reply integrity in orchestration paths (prevents corruption/overwriting during multi-step or multi-agent flows).  
  PR: https://github.com/elizaos/eliza/pull/8045

- Structural logic fixes and release contract fixes (CI/release pipeline reliability).  
  PRs: https://github.com/elizaos/eliza/pull/8071, https://github.com/elizaos/eliza/pull/8072

### CI / installation reliability
- **Global install smoke testing** was implemented to catch regressions when installing ElizaOS packages globally (common failure mode: wrong package paths / postinstall assumptions).  
  PR: https://github.com/elizaos/eliza/pull/8075

This pairs with recent issue triage around boot crashes and package path errors in `v1.7.2`:
- Issues: https://github.com/elizaos/eliza/issues/8000, https://github.com/elizaos/eliza/issues/8001

## 7) Breaking Changes / V1 → V2 Migration Warnings

### Breaking: Solana plugin relocation and rename
The move from `plugin-solana` → `plugin-wallet` under `chains/solana` is a **breaking structural change** for:
- import paths,
- documentation links,
- CI scripts that fetch plugins from prior locations,
- plugin discovery logic if you hard-coded the old name.

**Mitigation:** search/replace references to `plugin-solana` and update your repository automation accordingly.

### Token migration status (operational reminder)
A community question confirmed that **manual migration for pre-snapshot holdings is closed** after the Feb 4 deadline. This is not a code breaking change, but it may affect apps that still display migration CTAs or attempt portal flows.
