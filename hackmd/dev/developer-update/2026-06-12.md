# ElizaOS Developer Update (2026-06-06 → 2026-06-12)

This week focused on (1) unblocking autonomous multi-agent “monetized app” loops in the orchestrator, (2) hardening Cloud V2 authentication + routing (Steward + staging/prod isolation), (3) major plugin modularization (LifeOps split), and (4) continued Cloud Apps (Product 2) infra work (Terraform + per-tenant DB/container hosting).

---

## 1) Core Framework

### Orchestrator autonomy: child → parent dispatch via `USE_SKILL` broker
A key runtime capability gap was closed: sub-agents can now invoke the parent-agent broker dispatcher using `USE_SKILL`, enabling capped autonomous loops (e.g., build/deploy a monetized container app) without manual “human yes” prompts for every step.

- PR: **wire child→parent USE_SKILL dispatcher**: https://github.com/elizaos/eliza/pull/8275  
- Follow-on autonomy policies (spend allowance, economics capabilities): https://github.com/elizaos/eliza/pull/8250

**Why this matters:** previously, even if your sub-agent had the “economics” goal prompt, it could not reach the live broker path; now, the dispatch path is wired end-to-end.

### Agent view controllability (“agent-surface”) + view-action affinity
The agent runtime gained an **agent-surface framework** so plugin views can be enumerated/controlled by an agent, plus a **view-action affinity** layer that biases the planner toward actions relevant to the currently active view.

- PR: “Claude/agentic views surface”: https://github.com/elizaos/eliza/pull/8080

This is foundational for building operator-style UIs where the agent can reason about the UI state and choose view-scoped actions more reliably.

### Cloud agent web UI enabled by default
Cloud-hosted agents now ship with the web UI enabled by default (config hardening had reduced the original risk of unauthenticated exposure).

- PR: https://github.com/elizaos/eliza/pull/8167

### Community architecture note: local gRPC pre-filter pipeline for token-cost + latency control
In Discord, **carlgatz** described a local gRPC pipeline that filters high-volume blockchain/RPC data before hitting LLM parsing, keeps fast-path ops in RAM, and only escalates to MCP for deeper analysis—reporting sub-millisecond latency.

- Discussion (2026-06-11): https://discord.com/channels/1253563208833433701/1253563209462448241  
- Action item: seeking **5 devs** to stress test and document the design.

This is not merged code in ElizaOS, but it’s directly aligned with best practices for “LLM-last” architectures in high-throughput agents.

## 2) New Features

### Cloud API: container lifecycle surface (`/api/v1/containers`)
The Cloud API now exposes a first-class container deploy/read API and mutation verbs needed by autonomous app loops.

- Deploy + read route: https://github.com/elizaos/eliza/pull/8258  
- PATCH/DELETE/scale verbs: https://github.com/elizaos/eliza/pull/8270

**Example: create a container (conceptual request)**
```bash
curl -X POST "https://api.elizacloud.ai/api/v1/containers" \
  -H "Authorization: Bearer $ELIZA_CLOUD_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "image": "ghcr.io/your-org/your-app:latest",
    "env": { "NODE_ENV": "production" }
  }'
```

### Cloud SDK: web sign-in helper + `X-App-Id` affordance + STT helper
Third-party web apps integrating “Sign in with Eliza Cloud” and billed inference gained SDK support:

- Web sign-in helper + `X-App-Id` inference affordance: https://github.com/elizaos/eliza/pull/8251  
- `ElizaCloudClient.transcribeAudio()` helper for `/api/v1/voice/stt`: https://github.com/elizaos/eliza/pull/8266

**Example: transcribe audio via Cloud SDK**
```ts
import { ElizaCloudClient } from "@elizaos/cloud-sdk";

const client = new ElizaCloudClient({ baseUrl: "https://api.elizacloud.ai" });

const res = await client.transcribeAudio({
  file: new File([audioBlob], "speech.webm", { type: "audio/webm" }),
  model: "whisper-1" // model id depends on your org config
});

console.log(res.text);
```

### LifeOps modularization: monolith → focused plugins + `plugin-personal-assistant`
The large `@elizaos/plugin-lifeops` monolith was decomposed into a **personal assistant orchestrator** plus focused domain plugins (calendar, goals, inbox, finances, etc.). This is both an architectural and DX improvement: each domain now owns its actions/views, and cross-domain orchestration becomes explicit.

- PR (merged): https://github.com/elizaos/eliza/pull/8277  
- Calendar extraction (preceding step): https://github.com/elizaos/eliza/pull/8219

## 3) Bug Fixes (critical + high-impact)

### Orchestrator: session timeout correctly applied to ACP initialize handshake
ACP sub-agent sessions now honor the configured session timeout during the initial `initialize` JSON-RPC request (previously it always used a hardcoded default).

- PR: https://github.com/elizaos/eliza/pull/8286

**Impact:** prevents unexpected long hangs or mismatched timeout behavior in sub-agent bootstrap under custom session settings.

### Orchestrator on Windows: forward `Path` casing correctly
Environment forwarding to ACP sub-agents now correctly detects Windows path env vars (case sensitivity differences under Bun on Windows).

- PR: https://github.com/elizaos/eliza/pull/8285

## 4) API Changes (developer-facing)

### Auth transport changes for browser constraints (Cloud + agent web UI)
Browsers cannot set headers on `EventSource` (SSE) or during the WebSocket handshake. The agent server auth layer now supports token passing via query params for these transports:

- SSE: accept `?token=` on GETs: https://github.com/elizaos/eliza/pull/8309  
- WS: pass auth token via query param for handshake: https://github.com/elizaos/eliza/pull/8308

**Developer note:** if you embed ElizaOS UI in constrained browser contexts, ensure your client uses the updated connection strategy (UI client base was updated accordingly).

### Steward OAuth PKCE + signed nonce exchange (Cloud login)
Cloud login now sends PKCE on Steward OAuth and signs nonce-exchange upstream calls.

- PR: https://github.com/elizaos/eliza/pull/8259  
- Related: pin tenant via `X-Steward-Tenant` header: https://github.com/elizaos/eliza/pull/8179  
- Ongoing/open issue tracking prod login breakage: https://github.com/elizaos/eliza/issues/8257

**What to watch:** if you maintain custom Cloud frontend forks, you must include PKCE code challenge handling and the updated exchange signing behavior, or OAuth will 400/401.

### Cloud Apps (Product 2): app DB mode now explicit (`none|isolated`)
Apps no longer implicitly provision an isolated Postgres database for every deployment; database mode is now a first-class concept.

- PR: https://github.com/elizaos/eliza/pull/8338

If you rely on `DATABASE_URL`/`POSTGRES_URL` always being present, update your app assumptions and/or request isolated mode explicitly.

## 7) Breaking Changes (V1 → V2 migration warnings)

### `@elizaos/plugin-lifeops` removed (no shims)
The LifeOps monolith is **gone** and replaced by:
- `@elizaos/plugin-personal-assistant` (orchestrator role)
- new focused plugins: relationships, inbox, goals, blocker, finances, remote-desktop, plus expanded domain plugins

- Breaking refactor PR: https://github.com/elizaos/eliza/pull/8277

**Migration guidance:**
- Remove `@elizaos/plugin-lifeops` from your dependency graph.
- Replace imports/registrations with `@elizaos/plugin-personal-assistant` and whichever domain plugins you actually use.
- The `/lifeops` view was removed; use per-domain views instead.

### Cloud auth contract hardening (PKCE required)
Steward OAuth now requires PKCE and request signing in more places. If you’re on older Cloud frontend/app implementations, expect auth failures until you adopt:
- PKCE generation + verifier storage
- signed nonce exchange/refresh calls

- Fix PR: https://github.com/elizaos/eliza/pull/8259  
- Open issue (prod auth regression context): https://github.com/elizaos/eliza/issues/8257

### Transport auth changes (SSE/WS token query params)
If you have custom clients for SSE/WS, you may need to update to pass tokens via query params (since headers are not available in-browser for these handshakes).

- WS: https://github.com/elizaos/eliza/pull/8308  
- SSE: https://github.com/elizaos/eliza/pull/8309
