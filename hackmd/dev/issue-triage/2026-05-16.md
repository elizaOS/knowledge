## Patterns / Themes Suggesting Deeper Architectural Problems
- **“Merged with P1s outstanding” risk in Cloud paths:** Several critical behaviors were flagged in PR review (billing, auth codes, domain verification, internal auth). This indicates **insufficient pre-merge gating** for high-risk Cloud changes.
- **Silent failure modes on connector ingress:** Slack (and historically Telegram) demonstrate a recurring class: **exceptions on inbound message path cause silent drops**. In connectors, “never throw from event handler” should be a hard rule with standardized wrappers.
- **Split sources of truth (schemas/migrations):** The SQL migrator incident shows tooling allows drift without fast feedback. Similar drift risks can exist in config/auth detection and plugin enablement heuristics.
- **Operational-cost hazards from polling defaults:** Neynar credits issue suggests integrations can incur large unexpected costs without **rate/cost observability** and safe defaults.

---

## Process Improvement Recommendations
1. **Introduce “P0/P1 merge gates” for Cloud:** block merges when review flags P0/P1 unless explicitly waived with an owner + follow-up issue created automatically.
2. **Standard connector ingress wrapper:** a shared utility that:
   - catches/logs all handler errors,
   - records a “failed_ingress” memory/metric,
   - prevents silent drops,
   - applies consistent retry/backoff on rate limits.
3. **Schema drift CI check:** ensure DB migrations reflect all exported schema tables; fail CI on mismatch.
4. **Cost/Rate observability for external APIs:** standard metrics/log fields (`requests_per_hour`, `credits_estimate`, `poll_interval_effective`) + surface warnings in Settings UI when configured cost exceeds thresholds.
5. **Post-merge “risk burn-down” cadence:** weekly audit of merged PR review findings to ensure follow-ups are filed, prioritized, and closed (especially for Cloud/billing/security areas).
