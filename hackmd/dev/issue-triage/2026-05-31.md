## Patterns / Themes Suggesting Deeper Issues
- **Routing/contract drift between Stage 1 → planner → tool execution:** even when Stage 1 identifies the correct tool, the planner may ignore hints and choose unsafe/expensive fallbacks (SHELL). This indicates the system lacks a strong “hint authority” contract and guardrails for expensive tools.
- **User-facing leakage of internal orchestration control text:** sub-agent pipeline still risks exposing framework internals, which is both a UX and security-hardening concern (information disclosure).
- **Repo reorgs without migration affordances:** plugin moves (and other structural changes) are happening faster than documentation and discovery surfaces update, causing repeated support load.

## Process Improvements (Prevention)
1. **Add “Breaking Move” checklist for structural refactors:** required items: migration note, search-and-update templates, deprecation/redirect strategy, Discord announcement, and a CI grep to detect stale paths in docs.
2. **Introduce contract tests for planner correctness on common intents:** especially “search messages”, “run shell command”, “spawn sub-agent”, including negative checks (no unbounded recursive grep; no control metadata in replies).
3. **Establish an incident intake path for Discord-reported runtime issues:** lightweight form/template requiring image tag, logs, env vars, and reproduction steps; auto-file a GitHub issue label `from-discord`.
4. **Standardize user-facing output sanitization layer:** a shared “connector output policy” that strips control-plane artifacts regardless of which sub-agent/tool produced them.
