## Cross-Issue Patterns / Themes
- **“Go-live gaps” are mostly integration + operations, not core design:** The Apps plane architecture is sound, but missing/unfinished operational components (ingress, pooling, multi-node routing) are the real blockers.
- **Auth contracts drift between services:** Steward hardening (PKCE/HMAC/session requirements) can break Cloud flows unless there’s a formal compatibility contract and continuous canary tests.
- **Scaling bottlenecks are configuration-driven:** Defaults like Postgres connections, per-app network strategy, and pool sizes cap scale far earlier than CPU/RAM limits.
