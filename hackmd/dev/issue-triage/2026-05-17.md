# Cross-Issue Patterns / Themes

- **“Merged with known P1s” risk:** Multiple Greptile-flagged P1 issues appear to have landed in merged PRs, especially in Cloud monetization paths. This suggests insufficient “stop-the-line” enforcement for billing/security correctness.
- **Silent failure modes:** Slack message drops and Neynar credit blow-ups share a common theme: errors are either swallowed or not surfaced with actionable diagnostics (leading to user-perceived randomness/cost surprises).
- **State-machine drift / missing invariants:** Domain lifecycle (`active` vs `verified`) indicates state transitions aren’t enforced by invariant tests or type-level constraints.
- **Billing correctness not treated as transactional:** Chat credit reserve/reconcile needs stronger idempotency and failure semantics typical of payments systems.
