## Patterns / Themes Suggesting Deeper Architectural Issues

- **Release/packaging fragility:** Multiple signals point to **install/startup regressions** (#8000/#8001) that should be caught by standardized smoke tests.
- **Cross-cutting routing/orchestration complexity:** Sandbox routing (PR #8104) and web lookup regressions (PR #8067) suggest **tool/action routing contracts** are sensitive to refactors and security gating changes.
- **Ecosystem churn + migration risk:** Plugin relocation (Solana → plugin-wallet) and ongoing plugin additions increase the chance of **docs drift** and **breaking changes without clear migration paths**.
- **Observability gaps:** Discord “unspecified issue” highlights that problems are being raised without structured intake, making impact assessment and resolution slower.

## Recommendations: Process Improvements

1. **Add mandatory “Global Install + Boot” CI gate** for every release:
   - Install from the packed artifact (not workspace), run `eliza --version`, initialize a minimal project, and start a minimal agent.
   - Run on macOS/Linux/Windows and at least two Node LTS versions.
2. **Require an issue link for high-impact PRs** (routing/provisioning/security):
   - PRs like #8104/#8067 should have a canonical issue describing reproduction, expected behavior, and test plan.
3. **Adopt a “Migration Note” checklist for repo re-orgs**:
   - Any move/rename (e.g., plugin-solana → plugin-wallet) must include updated docs, a deprecation window (if possible), and searchable guidance.
4. **Improve intake from Discord → GitHub**:
   - Use a short triage form: environment, version, steps, logs, expected/actual.
   - Tag issues with `source:discord` and assign an initial owner within 24 hours.
5. **Expand regression tests around routing/tool contracts**:
   - Add contract tests for tool invocation, sandbox boundaries, and “web lookup” behaviors under security gating/feature flags.
