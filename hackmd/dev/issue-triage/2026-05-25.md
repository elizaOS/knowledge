## Patterns / Themes Suggesting Deeper Architectural Problems
- **Planner alignment drift:** Stage 1 classification can correctly suggest actions, but downstream planner selection overrides it (tool ranking/policy gap).
- **Missing “user-facing contract” enforcement:** Sub-agent control-plane metadata is leaking into end-user messages, implying insufficient separation between internal orchestration traces and connector output.
- **High-stakes agent capabilities scaling faster than safety baseline:** Economic/wallet agents are being shipped publicly while “secure-by-default” policies (caps, confirmations, injection hardening) aren’t standardized across the framework.
- **Platform/build strategy churn:** Large OS packaging shifts (and reversals) indicate unclear ownership and insufficient decision records for multi-arch + licensing constraints.

## Recommendations (Process Improvements)
1. **Introduce a “Stage 1 hint must be honored” rule** in planner policy, with automated tests for the top 20 user intents (search messages, summarize thread, find link, etc.).
2. **Create a strict internal/output boundary**:
   - tag all sub-agent/internal routing metadata as non-user-facing
   - enforce stripping at connector boundary
   - add snapshot tests ensuring no control headers leak.
3. **Add a “High-Stakes Capability Release Checklist”** (wallet access, trading, identity, deploy) requiring:
   - threat model + default-deny policy
   - spending limits + confirmation UX
   - prompt-injection regression tests.
4. **Adopt lightweight DDRs (decision records)** for major architecture reversals (OS build paths, multi-arch commitments) so CI gates and docs track the chosen direction.
