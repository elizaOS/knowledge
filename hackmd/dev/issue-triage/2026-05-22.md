## Conclusion

### 1) Top 5–10 highest priority issues to address immediately
1. **(P1)** v2.x **Getting Started + release notes** canonical docs (NEW)
2. **(P1)** **Official demos / YouTube / live examples** (NEW)
3. **(P1)** **Roadmap/version messaging clarity** (“v2 released” vs “v3 soon”) (NEW)
4. **(P2)** **Container/self-host reference architecture** for voice/GPU/latency (NEW)
5. **(P2)** **DeFi plugin validation + security posture** before promotion (NEW)
6. **(P2)** **Spec-decode hardening/regression tests** (M-RoPE/SWA+DFlash) (#7631/#7635 follow-up)
7. **(P2)** **Intel iGPU recommender guardrails** to avoid slow model routes (#7638 follow-up)
8. **(P3)** Document stance on **single-container “all-in-one” deployments** (NEW)

### 2) Patterns/themes suggesting deeper architectural problems
- **Release ≠ usability:** v2.x shipped, but **activation materials (docs/demos/roadmap clarity)** lag behind, creating a perception gap.
- **Silent degradation risks in local inference:** multiple recent issues revolve around **features disabling silently** (spec-decode fallbacks, performance cliffs). Even when fixed, this indicates a need for stronger capability detection and “fail loud” UX.
- **Deployment complexity pressure:** community is pushing toward simplified packaging (single-container), suggesting a broader need for **opinionated, supported deployment blueprints**.

### 3) Process recommendations to prevent recurrence
- **Add a release checklist gate:** no release announcement without (a) short release notes, (b) updated quickstart, (c) one validated demo path.
- **Introduce “capability/compat” contracts in CI:** lightweight tests that assert critical inference capabilities are either active or explicitly reported as inactive (no silent fallback).
- **Establish plugin promotion criteria:** verified/unverified labeling, minimum security checks, and compatibility smoke tests before highlighting in official channels.
- **Maintain a pinned “State of the Project” artifact:** updated each release, referenced by Discord mods to reduce repeated onboarding churn.
