# Patterns / Themes Suggesting Deeper Issues

- **Auth contract drift across services (Cloud ↔ Steward):** PKCE, signing/HMAC, tenant scoping, and redirect rules are changing and repeatedly breaking production/staging flows. This indicates missing **versioned integration contracts** and insufficient end-to-end synthetic monitoring.
- **Environment skew (staging vs prod) and deployment coupling:** Multiple fixes referenced across PRs imply frequent “works in staging, fails in prod” or vice versa; suggests gaps in **config parity** and **runtime validation**.
- **Large refactors/features landing alongside critical infra changes:** Big UI/orchestrator and plugin refactors increase regression surface; without aggressive contract tests, user-facing breakage appears as “random agent quality issues.”

# Process Improvement Recommendations

1. **Add production synthetics for critical user journeys** (login via OAuth, email magic link send/verify, token refresh) with alerting and fast rollback guidance.
2. **Introduce contract tests between cloud-api and Steward**:
   - PKCE requirements, required headers/signatures, allowed redirect URIs, tenant routing behavior.
3. **Adopt “integration change checklist” for auth/security changes**:
   - Update frontend, worker proxy, and upstream simultaneously; include staged rollout plan.
4. **Strengthen regression test coverage for orchestrator output fidelity**:
   - Golden tests for “deliverable must appear verbatim” across model backends; enforce “terminal attempt only” rendering.
5. **CI preflight for release workflows and secrets**:
   - Avoid “workflow startup_failure” by validating required secrets inside the workflow and gating platform jobs explicitly.
