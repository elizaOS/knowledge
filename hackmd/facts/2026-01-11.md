# Fact Briefing: 2026-01-11

## Overall Summary
The ElizaOS ecosystem is advancing toward a decentralized future with the announcement of the Jeju layer and Eliza 2.0, while simultaneously navigating a regulatory crisis in the Korean market. Technical progress in multi-language support (Rust, Python) and LLM-free agent architectures is accompanied by community debates regarding tokenomics and migration transparency.

## Key Facts

- Jeju, a new ecosystem layer focusing on agent activity and utility, is targeted for launch in H2 2026.
- ElizaOS has been delisted from major Korean exchanges including Bithumb, Coinone, and Korbit due to transparency issues.
- Shaw demonstrated Eliza agent implementations in Rust and Python using fewer than 50 lines of code.
- A working branch for Eliza V2.0.0 (PR #6351) was opened, featuring a multi-language runtime and removal of non-essential CLI components.
- ElizaCloud weather data API integration is enabled and the first cloud payment transactions have been completed.
- Eliza development this week increased documentation coverage from 60% to approximately 95%.
- DAXA terminated exchange support for ai16z/elizaOS citing deficiencies in the migration process disclosure.
- A novel WoW anti-detection technique was developed using pixel encoding of game API calls decoded at 200 screenshots per second.
