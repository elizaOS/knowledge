# Fact Briefing: 2026-02-14

## Overall Summary
ElizaOS is transitioning to a multi-language architecture supporting Rust and Python while implementing significant security hardening via JWT authentication. Community attention is divided between these technical advancements and critical UX gaps regarding non-technical user onboarding and token utility clarity.

## Key Facts

- Next-generation support for Rust and Python alongside TypeScript was introduced in elizaos/eliza PR #6485.
- A global JWT authentication system was launched in elizaos/eliza PR #6200 to enable data isolation and secure user sessions.
- A Moltbook verification solver using the DeepSeek-chat model was implemented to automate math-based anti-bot challenges.
- The ActionFilterService was introduced to optimize AI performance by using vector search and BM25 reranking to prune prompt bloat.
- Standardized blockchain integration was achieved by connecting the EVM and Solana plugins to the Spartan-Intel chain registry.
- The n8n-workflow plugin was updated to version 1.1.0, enabling agents to modify and redeploy existing workflows.
- SIWE (Sign-In With Ethereum) authentication is being added to ElizaCloud to allow agents to generate API keys autonomously.
- Developers consolidated duplicate n8n repositories, selecting plugin-n8n-workflow as the primary repository due to superior features.
