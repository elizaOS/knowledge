# Fact Briefing: 2026-06-18

## Overall Summary
The ElizaOS project has completed a major organizational milestone by consolidating all development work into a single repository and transitioning to a modular 'Cloud Apps' infrastructure. However, the community is actively debating the project's long-term open-source strategy and managing a surge in migration-related scam activity.

## Key Facts

- Development work has been consolidated into the single eliza repository as of June 15, 2026.
- The Eliza Cloud Apps data-plane was launched, implementing per-tenant container isolation and automated environment provisioning.
- Secure Blockchain Development Corp. has partnered with the Eliza Foundation to commercialize enterprise-level AI agents built on the elizaOS framework.
- Odilitime granted minimod powers to om1d_sa on June 17 to address persistent wallet and airdrop scams in the #coders channel.
- A collaboration proposal for Semantic Delegation using ERC-7710 totaling $7,500 USDC in funding was presented by Saulodigital.
- Steward login functionality in production was identified as broken due to a PKCE code_challenge mismatch, blocking Eliza Cloud sign-ins.
- The official AI16Z to ELIZAOS token migration window has closed, with no legitimate alternative migration links available for those who missed the deadline.
- Eliza Cloud architecture was refactored to allow agents to manage economic tasks by delegating skills to sub-agents.

## Open Questions

- Is there any alternative process for migrating legacy AI16Z tokens after missing the official deadline?
- What are the specific details of the commercialization partnership between Secure Blockchain Development Corp. and the Eliza Foundation?
