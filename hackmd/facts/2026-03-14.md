# Fact Briefing: 2026-03-14

## Overall Summary
Development has shifted towards the Eliza 2.0.0 alpha release and core architecture refactoring, while the community remains focused on security vigilance following the closure of the AI16Z to elizaOS token migration.

## Key Facts

- The AI16Z to elizaOS token migration officially closed on February 4, 2026, after a three-month window.
- Odilitime published a runtime refactor proposal on HackMD with a feedback deadline of Sunday, March 15.
- Eliza 2.0.0 alpha version was published, marking a transition to the framework's next major iteration.
- The Milady app launch is targeted for approximately March 26, 2026.
- A critical race condition was identified in plugin-sql where the database adapter registers as a side-effect during parallel initialization.
- The Babylon platform launched its ticker service to an initial user base of 50,000.
- Aiprox.dev implemented an autonomous agent registry featuring a usage-based rating system and multi-payment rail support.
- Moderators identified and flagged a phishing scam involving a fake support bot directing users to a 'colabdesk' site requesting seed phrases.

### Strategic Insights

#### Value Accrual Pivot
The project is shifting toward a cloud-service business model where profits from peripheral apps (like Milady) are used for elizaOS token buybacks rather than direct utility.

*Implications/Questions:*
  - How will the community react to the lack of direct token utility inside new apps?
  - Will the buyback mechanism be sufficient to offset current selling pressure?

#### Security by Isolation
The adoption of Spartan infrastructure to isolate LLMs from private keys reflects a hardening of the security posture for agent-led financial transactions.

*Implications/Questions:*
  - Should this isolation be a mandatory standard for all plugins in the registry?
