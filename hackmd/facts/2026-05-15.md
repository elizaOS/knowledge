# Fact Briefing: 2026-05-15

## Overall Summary
The ecosystem is focused on AI agent model optimization through a new Eliza-1 (Qwen3.5) series and the introduction of a decentralized 'Pay or Burn' escrow protocol on Base. Infrastructure efforts are centered on the Eliza v3 transition and resolving plugin registry accessibility issues.

## Key Facts

- Zadayos announced the Eliza-1 model series (Qwen3.5) with parameter sizes ranging from 0.6B to 27B-1M.
- A 'Pay or Burn' escrow protocol for AI agents was deployed on Base Mainnet, utilizing game theory to handle disputes without human arbitration.
- The v2 plugin registry infrastructure (plugins.elizacloud.ai) is reporting 404 errors, causing a potential rollback to direct monorepo PRs.
- ElizaOS confirmed compatibility with the GODL protocol's skill.md format for autonomous mining and staking on Solana.
- Shaw reported development of a fine-tuning pipeline for Qwen models to enhance action calling and planning tasks.
- Neynar API users reported credit consumption spikes of 300 credits/hour despite 120-second polling overrides, with webhooks suggested as a fix.
- The @elizaos/vault package was introduced to provide cross-platform AES-256-GCM encryption for sensitive configuration data.
- The Jupiter integration was migrated to the official api.jup.ag endpoint with mandatory API key authentication requirements.
