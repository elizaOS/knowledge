### Strategic Insights

#### Korean exchange delisting drives urgent communications requirements
Korean exchanges scheduled the removal of AI16Z/ElizaOS trading and later deposits/withdrawals while citing rebrand/swap transparency, and Discord users expressed confusion about auto-migration on exchanges. A single authoritative migration and exchange-handling statement could reduce conflicting community guidance.

*Implications/Questions:*
  - Is there an official exchange-by-exchange migration policy statement that can be published and pinned?
  - What disclosure artifacts (timeline, snapshot rules, swap mechanics) can be made explicit to address transparency concerns?

#### Versioning and compatibility management between core and plugins
Discord integration failures tied to the serverId‚ÜímessageServerId migration, plus the need to test plugin-discord compatibility across branches, show that core/plugin release coordination affects user-facing reliability. Merged fixes (e.g., #6333) and pending release blockers in plugin-discord reinforce the need for clear compatibility matrices and release automation health.

*Implications/Questions:*
  - Should releases include a published compatibility table (core version ‚Üî plugin versions)?
  - Is the plugin-discord publishing pipeline issue (#40) the highest operational blocker for Discord stability?

#### Data products need last-mile integration surfaces
Core-dev discussion emphasized that agentic workflows already generate high-quality daily/weekly/monthly insights, but delivery into agents/webhooks/apps remains a gap. Prioritizing integration endpoints and reference apps could convert internal insights into user-visible value.

*Implications/Questions:*
  - Which integration surface (webhooks vs in-product UI vs agent API) is the fastest path to shipping these insights?
