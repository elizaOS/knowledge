# Fact Briefing: 2026-04-30

## Overall Summary
ElizaOS is undergoing a significant architectural shift by decoupling the core agent server from application packages while simultaneously expanding native mobile capabilities for Android. Core developer Shaw addressed community concerns regarding market performance by confirming active work on Eliza Cloud and a token utility model via the x402 payment standard.

## Key Facts

- The core agent server was decoupled from application-specific packages using a new plugin-lifecycle registration system on April 29.
- Native local agent architecture for Android was implemented, featuring a foreground service for Bun runtime execution.
- The $ELIZA token has been implemented as the default payment method in x402 for elizaOS and Milady infrastructure services.
- Eliza Cloud infrastructure was migrated from Next.js to a Vite SPA with Hono Workers, utilizing R2 for image generation storage.
- A major technical modernization was completed including upgrades to TypeScript, Node.js, and Bun across the repository monorepo.
- Support for seven different blockchain networks was integrated into the plugin registry to enable multi-chain payments.
- A new NoModelProviderConfiguredError was introduced to the framework to provide actionable feedback for agent misconfigurations.
