# Fact Briefing: 2026-06-13

## Overall Summary
The project is currently focused on stabilizing the Eliza Cloud Apps infrastructure and addressing critical 'Eliza Cloud' signup blockers while exploring strategic expansions into decentralized robotics. Engineering efforts are concentrated on resolving P0 scaling and ingress gaps identified in recent infrastructure audits.

## Key Facts

- Users reported a technical blocker receiving the error 'Tenant elizacloud requires an invitation to join' during signup on June 12.
- PRs #8394 and #8393 were completed on June 12 to enable staging node ingress via Caddy and automate Headscale installation.
- A proposal was submitted by valorenvk to create a native @elizaos/plugin-peaq to integrate elizaOS as a reasoning framework for 3 million IoT devices.
- A local gRPC pipeline was developed by carlgatz to optimize AI token processing, achieving sub-millisecond latency during high-volume spikes.
- Issue #8321 remains a significant blocker for the apps-data-plane go-live due to a state-provider mismatch involving orphaned random_password resources.
- PR #8277 was merged to decompose the 'plugin-lifeops' monolith into six specialized modular plugins, improving framework flexibility.
- Infrastructure audits identified that the current Eliza Cloud Apps code supports dozens of concurrently running apps, but requires P0 fixes to reach the 1000-app target.
- Automatic billing and metered usage tracking were introduced to support the transition to a professional-grade cloud-hosted model.
