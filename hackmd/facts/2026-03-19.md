# Fact Briefing: 2026-03-19

## Overall Summary
The ElizaOS ecosystem is transitioning into a high-velocity launch phase with the imminent release of the Milady app and Babylon platform, alongside significant new infrastructure for decentralized agent persistence. However, the team is facing critical community pressure regarding token performance and transparency issues related to the ai16z migration supply.

## Key Facts

- Odilitime confirmed elizacloud is currently generating revenue and hosting internal product development.
- The Milady app and Babylon platform are scheduled for launch by early next week, with Milady potentially launching as early as March 18 evening.
- The Ensoul persistence plugin (@ensoul-network/plugin-elizaos) was released, providing decentralized, encrypted storage for agent consciousness via erasure coding.
- Internal framework refactoring renamed ‘plugin-forms’ to ‘plugin-form’ and ‘plugin-form’ to ‘plugin-form-chain’ to standardize naming conventions.
- PR #6597 was introduced to implement a skills folder structure for ElizaOS v2.0.0, proposing a decentralized discovery model with zero default skills.
- Dinesh integrated 100+ onchain data supports via goldrush.dev and implemented fixes for the plugin-evm component.
- Community members reported that approximately 54% of the elizaOS total supply designated for ai16z holders remains unaccounted for due to a low migration rate.
- Effect AI proposed a new API integration to allow ElizaOS agents to post tasks to a decentralized human worker network for human-in-the-loop intervention.
