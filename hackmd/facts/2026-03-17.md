# Fact Briefing: 2026-03-17

## Overall Summary
Development has pivoted toward elizaOS v2.0.0 architectural decisions and multi-chain data integration, while the community is demanding immediate transparency regarding the ai16z to elizaOS token migration analytics.

## Key Facts

- Dinesh announced the addition of 100+ onchain data support via goldrush.dev integration for the plugin-evm component.
- Odilitime introduced PR #6597 for v2.0.0, implementing a new skills folder structure to the core framework.
- The v2.0.0 architectural proposal suggests shipping with zero default skills to prevent the 'plugin bloat' seen in version 0.x.
- Community analysis reported approximately 100,000 unique addresses still hold ai16z tokens, suggesting only a 5-10% migration rate to elizaOS.
- Effect AI proposed an API integration to allow ElizaOS agents to post tasks to a decentralized human worker network.
- The x402Guard security proxy was introduced to provide non-custodial spend limits and contract whitelisting for DeFi agents on Base and Solana.
- OpenClaw reported sold-out Mac Mini hardware in China, driven specifically by demand for dedicated AI agent hosting.
- The SAID Protocol was integrated to provide digital identities for agents on the Solana blockchain.
