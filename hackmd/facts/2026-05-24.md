# Fact Briefing: 2026-05-24

## Overall Summary
elizaOS successfully released version 2.0.2, pivoting towards an 'Owner+Agent' identity model while hardening cloud and mobile infrastructure. Concurrently, community discussions centered on the closure of the Babylon project and the end of the ai16z-to-elizaOS token migration period.

## Key Facts

- elizaOS v2.0.2 was released, officially introducing the OWNER+AGENT identity architecture and job-queue based lifecycle management.
- The migration period from ai16z to ElizaOS has officially ended and the migration team has moved on.
- The Babylon project was shut down due to lack of funding, with its training simulations integrated into elizaOS under the name 'Feed'.
- Highest liquidity for ElizaOS tokens is currently available on the Binance Smart Chain (BSC).
- A new agentic launchpad called waifu.fun was launched, built using elizaOS v3 agents and developed by an agent named Sol (The Architect).
- Technical implementation began for `@elizaos/vault`, a cross-platform secrets management system to replace plaintext storage in config files.
- Developers are actively addressing performance cliffs (1.15 t/s) for Qwen2.5-14B models on Intel Lunar Lake iGPUs.
- Support for OAuth was enabled for X, Linear, Calendly, and Shopify to support the new identity model in PR #7851 and #7820.
