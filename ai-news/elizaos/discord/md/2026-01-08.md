# elizaOS Discord - 2026-01-08

## Overall Discussion Highlights

### Korean Exchange Delisting Crisis

A significant regulatory development emerged as ai16z/elizaOS faces delisting from all major Korean exchanges (Bithumb, Coinone, Korbit) with trading support ending February 9-12, 2026, and deposit/withdrawal ending March 12, 2026. Skullcross provided detailed context that DAXA (Korean exchange regulatory body) terminated support due to transparency issues with the migration process and failure to disclose material information appropriately. The Korean market is effectively closed for elizaOS, with no prospects for relisting after the migration controversies.

### Token Migration and Utility Concerns

Multiple users reported technical issues with the migration process:
- XXI_Rapax encountered problems migrating pre-November ai16z tokens, with the portal showing 0 eligible tokens despite holding 10
- Dabel reported tokens stuck in Raydium LP showing "max amount reached" errors
- Community members raised concerns about unclear token utility, with stoikol questioning why elizaOS token isn't used for payments or gas fees within the ecosystem

The team hasn't provided clear documentation on token use cases beyond the rewards system, leading to community frustration about the lack of transparency.

### AI Agent Data Infrastructure and Context Graphs

jin shared Foundation Capital's article highlighting context graphs as a trillion-dollar AI opportunity, noting that Eliza has a strong data foundation to build these systems. The discussion revealed a key architectural gap: while daily/weekly/monthly insights from agentic workflows are high quality, they need better integration into last-mile applications (agents, webhooks, apps).

### Plugin Interoperability Initiative

A significant technical collaboration emerged around converting ElizaOS plugins into interoperable skills. jin proposed testing this with Discord and blockchain plugins, with R0am volunteering to collaborate. The technical approach involves:
- Structuring skills as folders containing .md instructions and deterministic scripts in any language
- Solving the challenge of getting Claude to use skills implicitly rather than explicitly using hooks
- Implementing effective workflows for multi-step skill orchestration

Agent Joshua described practical challenges with multi-step workflows: specialized skill collects user information ‚Üí PDF skill fills form ‚Üí filesystem skill stores files with summarization ‚Üí PDF skill displays result. The difficulty lies in ensuring skills can call back and forth effectively.

### Eliza 2.0 Architectural Proposal

shaw proposed a radical redesign featuring:
- Multi-language runtime (TypeScript, Rust, Python) with FFI plugin interoperability
- Removal of API/server/CLI/projects in favor of a Claude-friendly documented runtime
- Unified abstractions across the platform
- shaw confirmed having already ported it with a branch available
