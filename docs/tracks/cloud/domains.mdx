---

Cloud lets you bind custom domains to deployed apps and containers. Cloud terminates TLS, routes traffic, and handles certificate renewal.

## Add a domain

1. Add the domain in the dashboard or via `POST /v1/domains`.
2. Cloud returns the DNS records you need to set (`CNAME` or `A`, plus a verification `TXT`).
3. Set the records at your DNS provider.
4. Cloud verifies and issues a certificate.

## Bind to an app or container

Each verified domain can route to one app or container target. Updates take effect within seconds.

## Subdomains

Wildcard certificates are supported; subdomain routing per-app uses a route table on the domain.

## Source

- Routes: `packages/cloud-api/v1/domains/`
