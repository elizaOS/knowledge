Eliza se configura a travĂ©s de un archivo `eliza.json` almacenado en el directorio
de estado (predeterminado: `~/.eliza/eliza.json`). Todas las secciones son opcionales; Eliza
usa valores predeterminados razonables cuando se omiten los campos.

El enrutamiento canĂłnico en tiempo de ejecuciĂłn se persiste de forma separada de los campos
de compatibilidad heredados. La fuente de verdad de nivel superior en tiempo de ejecuciĂłn es:

- `deploymentTarget`
- `linkedAccounts`
- `serviceRouting`

La configuraciĂłn se valida contra esquemas Zod al momento de la carga. Los campos invĂˇlidos producen advertencias pero no bloquean el inicio.

## Enrutamiento canĂłnico en tiempo de ejecuciĂłn
</div>

Estos tres campos definen la topologĂ­a activa en tiempo de ejecuciĂłn:

| Campo | DescripciĂłn |
|------|-------------|
| `deploymentTarget` | DĂłnde se ejecuta el servidor activo: `local`, `cloud` o `remote` |
| `linkedAccounts` | QuĂ© cuentas de proveedor o nube estĂˇn vinculadas |
| `serviceRouting` | QuĂ© backend maneja `llmText`, `tts`, `media`, `embeddings` y `rpc` |

```json5

    runtime: "remote",
    provider: "remote",
    remoteApiBase: "https://eliza.example.com",
  },
  linkedAccounts: {
    elizacloud: {
      status: "linked",
      source: "oauth",
    },
  },
  serviceRouting: {
    llmText: {
      backend: "openrouter",
      transport: "direct",
      primaryModel: "openrouter/anthropic/claude-sonnet-4.6",
    },
  },
}
```
